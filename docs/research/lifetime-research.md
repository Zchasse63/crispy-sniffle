## 2. POLICY PATTERNS — Real Guest/Drop-In Models Observed

Four distinct access models emerged, and they vary by club maturity and metro more than by a formal tier.

**Model A — Hybrid (public day pass + members):** the most common. A dedicated daypass page lists a concrete price, applied to first month's dues if you join. Observed prices clustered tightly by metro/positioning:
- $50 — North Scottsdale
- $60 — Sandy Springs, Las Colinas, Potomac
- $75 — Buckhead, Westlake, Chestnut Hill
- $100 — One Wall Street, Sky (Manhattan), PENN 1 *(all NYC)*
- ~$100 reported-but-unverified — Walnut Creek

**Model B — Inquiry-driven / members-first (no public day-pass price):** Palm Beach Gardens, Boca Raton, Coral Gables, Miami at The Falls, Frisco. Starting monthly shown ($329–$339) but day-pass/guest price gated.

**Model C — Members-only + waitlist (exclusive clubs):** Harbour Island (reported 10,000+ waitlist at opening, members-only, no public day pass, *no* monthly price published), West Boca (newly opened, "Join the Waitlist" + "Before Summer Pricing," all prices hidden), and the NYC Premier tier (Sky, PENN 1) which is explicitly waitlist-only for indoor pickleball.

**Model D — Member-invited guest (corporate, applies chain-wide):** member invites via the Life Time app → guest downloads app, creates free account, accepts invite, shows photo ID, checks in via in-app QR; **same guest once per 60 days**; adults 18+ register; one-day membership required for ages 14+ when no invite (13-and-under complimentary).

## 3. DATA-MODEL IMPLICATIONS FOR SCOUT

### Fields to add (with the variant structure the data demands)

| Field | Type / values | Notes |
|---|---|---|
| `monthly_price_starting` | int, nullable | "Starting at" floor only; **not** tier dues. Null when waitlist/inquiry-gated. |
| `monthly_price_is_starting_floor` | bool | Flag that the number is a floor, not full dues. |
| `day_pass_price` | int, nullable | The scrapable public number. |
| `day_pass_credited_to_dues` | bool | Consistently true where a day pass exists. |
| `day_pass_verified` | enum: `page` / `third_party` / `unconfirmed` | Critical — Walnut Creek/Frisco/Boca day-pass figures came from search snippets, not the page. Don't present unverified as fact. |
| `enrollment_fee` | int, **nullable — expect null** | Never scrapable (0/18). Must be owner/manual. |
| `guest_policy_model` | enum: `public_day_pass` / `member_invite_only` / `members_only_waitlist` / `hybrid` | Captures Models A–D. |
| `guest_pass_60day_limit` | bool | Corporate default; applies chain-wide. |
| `app_required_for_guests` | bool, nullable | Corporate policy; not per-club. |
| `waitlist` | enum: `none` / `url_exists_unconfirmed` / `active_confirmed` | Several clubs have a waitlist *URL* but render normal copy — don't treat URL existence as an active waitlist. |
| `tier_label` | string, **nullable — expect null** | Never printed publicly. "Signature" is a *membership type*, not a club tier. Marketing terms ("Athletic Country Club," "Luxury Health Club") are not formal tiers. |
| `segment` | const: `luxury` | All 18. |
| `data_source` | enum: `official_page` / `corporate_policy` / `third_party` / `owner_manual` | Provenance per field. |

### Facts that are NOT publicly scrapable → must come from owner/manual

- **Enrollment / initiation fee** (0/18 — the single biggest gap).
- **Full tier-by-tier dues** (only a starting floor is ever public).
- **Formal internal tier name** (Diamond/Signature/Exclusive/Athletic).
- **Per-club guest-policy specifics** (hours, accompaniment rules at the club level).
- **Day-pass/guest price for ~half the clubs** (Model B/C — gated).
- **Whether a "waitlist URL" is an actually-enforced cap.**

**Modeling guidance:** make `enrollment_fee`, `tier_label`, and full-dues fields nullable-by-default and treat them as an *owner-claim/manual-enrichment* layer. Stamp every economic field with provenance so the UI can distinguish "official page" from "third-party/unconfirmed." Guest mechanics should be modeled once as a **chain-level corporate default**, overridable per club, rather than scraped per location.
